# P002 - Axes, frames, ticks, and labels in GSP output semantics

Status: pending ChatGPT Pro response.

## Why This Is Needed

The repository has two different axes models:

- Legacy `src/vispy2/axes/*` code builds axes as ordinary legacy GSP visuals
  (`Segments`, `Texts`) and appends them to the same render stream as plotted data.
- The current M008 VisPy2 MVP intentionally emits only formal v0.1 protocol
  `PointVisual` and `ImageVisual` objects, and excludes axes, ticks, labels,
  transforms, styling, and layout.

This creates an architecture question: should axes/ticks/labels be represented as
semantic protocol objects, expanded by a producer into ordinary visual primitives,
generated by each backend from viewport state, or treated as non-queryable overlay
decoration? The answer affects protocol conformance, VisPy2 public API, Matplotlib
reference behavior, Datoviz feasibility, query/readback semantics, and future remote
rendering.

### Query Semantics Excerpt

```markdown
Use a unified panel query:
"What rendered scene contribution is under this panel coordinate?"

First-slice QueryResult:
- request id;
- status;
- hit/miss;
- panel/framebuffer coordinate;
- visual id;
- visual family;
- item/texel id where applicable;
- visual/data coordinate where available;
- displayed RGBA;
- optional scalar/vector/category/text value.

Statuses:
- `hit`: A visual contribution was found.
- `miss`: Query coordinate is inside the panel but no visual contains it.
- `outside-panel`: Query coordinate is outside the panel/viewport bounds.
- `unsupported`: Backend cannot answer the requested query mode or payload.
- `stale`: Result violates freshness policy.
- `dropped`: Request was cancelled/superseded/dropped.
- `failed`: Backend attempted query/readback but failed operationally.

Capabilities advertise coarse query support through `CapabilitySnapshot.query_modes`.
Current names:
- `panel-query`;
- `point-item`;
- `image-texel`.

When multiple visuals contain the coordinate, the reference proof chooses the highest
`z_order` entry as the frontmost result.
```

class Figure:
    axes: list["Axes"] = field(default_factory=list)

    def add_axes(self) -> "Axes":
        axes = Axes(figure=self)
        self.axes.append(axes)
        return axes

    def visuals(self) -> tuple[PointVisual | ImageVisual, ...]:
        return tuple(visual for axes in self.axes for visual in axes.visuals)

    def render_matplotlib(self) -> tuple[Any, Any]:
        import matplotlib.pyplot as plt

        fig, mpl_axes = plt.subplots()
        for visual in self.visuals():
            if isinstance(visual, ImageVisual):
                render_image_visual(mpl_axes, visual)
            elif isinstance(visual, PointVisual):
                render_point_visual(mpl_axes, visual)
            else:
                raise TypeError(f"unsupported protocol visual: {type(visual)!r}")
        mpl_axes.set_aspect("auto")
        return fig, mpl_axes

    def savefig(self, path: str | Path, **kwargs: Any) -> None:
        fig, _ = self.render_matplotlib()
        try:
            fig.savefig(path, **kwargs)
        finally:
            import matplotlib.pyplot as plt
            plt.close(fig)


@dataclass(slots=True)
class Axes:
    figure: Figure
    visuals: list[PointVisual | ImageVisual] = field(default_factory=list)

    def scatter(self, x, y=None, *, c=None, color=None, s=36.0, size=None, id=None) -> PointVisual:
        positions = _positions(x, y)
        colors = _colors(c if c is not None else color, positions.shape[0])
        sizes = _sizes(size if size is not None else s, positions.shape[0])
        visual = PointVisual(id=id or _visual_id("points"), positions=positions, colors=colors, sizes=sizes)
        self.visuals.append(visual)
        return visual

    def imshow(self, image, *, extent=None, origin=ImageOrigin.UPPER,
               interpolation=ImageInterpolation.NEAREST, id=None) -> ImageVisual:
        image_array = np.asarray(image)
        if image_array.dtype == np.dtype(np.float64):
            image_array = image_array.astype(np.float32)
        if extent is None:
            height, width = image_array.shape[:2]
            if _origin(origin) == ImageOrigin.UPPER:
                extent = (-0.5, width - 0.5, height - 0.5, -0.5)
            else:
                extent = (-0.5, width - 0.5, -0.5, height - 0.5)
        visual = ImageVisual(
            id=id or _visual_id("image"),
            image=image_array,
            extent=extent,
            origin=_origin(origin),
            interpolation=_interpolation(interpolation),
        )
        self.visuals.append(visual)
        return visual


def subplots() -> tuple[Figure, Axes]:
    fig = Figure()
    ax = fig.add_axes()
    return fig, ax

        if model_matrix is None:
            model_matrix = Bufferx.mat4_identity()
        if camera is None:
            camera = Camera(Bufferx.mat4_identity(), Bufferx.mat4_identity())
        render_item = RenderItem(self._inner_viewport, visualBase, model_matrix, camera)
        self._render_items.append(render_item)
        return render_item

    def render(self) -> None:

        axes_transform_numpy = self._axes_display.get_transform_matrix_numpy()

        # Apply axes transform to all data visuals.
        for render_item in render_items:
            model_matrix_numpy = np.eye(4, dtype=np.float32)
            model_matrix_numpy = axes_transform_numpy @ model_matrix_numpy
            render_item.model_matrix = Bufferx.from_numpy(np.array([model_matrix_numpy]), BufferType.mat4)

        render_items_axes = self._axes_display.get_render_items()
        render_items_combined = render_items + render_items_axes

        self._renderer_base.render(
            [render_item.viewport for render_item in render_items_combined],
            [render_item.visual_base for render_item in render_items_combined],
            [render_item.model_matrix for render_item in render_items_combined],
            [render_item.camera for render_item in render_items_combined],

### Legacy Tick Locator and Formatter

```python
class AxisTickLocator:
    def __init__(self, target_ticks: int = 6, nice_fractions=(1, 2, 2.5, 5, 10)) -> None:
        self.target_ticks = target_ticks
        self.nice_fractions = nice_fractions

    def tick_step(self, min_dunit: float, max_dunit: float) -> float:
        if min_dunit == max_dunit:
            return self._fallback_step(min_dunit)
        data_range = abs(max_dunit - min_dunit)
        raw_step = data_range / max(1, self.target_ticks)
        exponent = math.floor(math.log10(raw_step))
        base = 10**exponent
        fraction = raw_step / base
        for nice in self.nice_fractions:
            if fraction <= nice:
                return nice * base
        return self.nice_fractions[-1] * base

    def compute_location_dunit(self, min_dunit: float, max_dunit: float) -> tuple[list[float], float]:
        tick_step = self.tick_step(min_dunit, max_dunit)
        start = math.ceil(min_dunit / tick_step) * tick_step
        end = math.floor(max_dunit / tick_step) * tick_step
        tick_count = int(round((end - start) / tick_step)) + 1
        tick_positions = [start + tick_index * tick_step for tick_index in range(max(0, tick_count))]
        tick_positions = [self._round_tick(tick_position, tick_step) for tick_position in tick_positions]
        return tick_positions, tick_step


class AxisTickFormatter:
    def __init__(self, scientific_threshold: int = 3) -> None:
        self.scientific_threshold = scientific_threshold

    def precision(self, step: float) -> int:
        if step <= 0:
            return 0
        return max(0, -math.floor(math.log10(step)))

    def format(self, tick_value: float, tick_step: float) -> str:
        if tick_value == 0:
            return "0"
        magnitude = abs(tick_value)
        order = math.floor(math.log10(magnitude))
        if abs(order) >= self.scientific_threshold:
            return f"{tick_value:.0e}"
        decimals = self.precision(tick_step)
        return f"{tick_value:.{decimals}f}"
```

### Current Decision Tension

The old code proves that axes can be regenerated as ordinary line/text visuals and
sent through the rendering path, which is attractive for backend uniformity and for
Datoviz-like renderers. However, doing only that may erase semantic information:
after expansion, the protocol/server sees line segments and glyphs, not an x-axis,
tick values, data limits, label roles, query policy, controller state, or layout
intent. The new MVP avoided this by not implementing axes yet, but VisPy2 needs axes
soon to be useful.

## Dependent Work To Pause

Do not implement protocol-level axes semantics until this decision is answered and
captured in an ADR. In the meantime, it is acceptable to keep the M008 MVP as
point/image-only and to mine legacy axes code for algorithms and examples.
