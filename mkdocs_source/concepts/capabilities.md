# Capabilities and adaptation

A capability snapshot describes what one concrete session can do. Support is evaluated for a
specific feature and scope, not for a backend name in general.

| State | Meaning |
|---|---|
| Strict | The stated semantics have current conformance evidence. |
| Adapted | A documented approximation or semantic deviation is used. |
| Partial | Only an enumerated subset is supported. |
| Unsupported | The feature is rejected or not advertised. |
| Blocked | Promotion is intentionally withheld pending correctness or stability evidence. |
| Not assessed | No reliable claim has been made. |

API symbols and screenshots are useful evidence, but neither is sufficient for a strict claim.
Runtime capability discovery remains authoritative.
